# LongNovelInsight v0.3.0 — EPUB, Search & Evidence

> **Release Date:** 2026-05-31
> **Status:** Released. All planned features implemented and verified.
> **Tag:** `v0.3.0`

---

## Goals

Add EPUB support, SQLite FTS5 full-text search, hybrid retrieval across multiple source types, structured chat evidence, entity evidence explorer, and similar scenes — all fronted by a clean React UI. Enable users to analyze both TXT and EPUB novels with precise source-tracking to the exact chapter and chunk, and debug retrieval quality with a built-in trace drawer.

### What was delivered

**Backend (12 steps):**
- EPUB upload/parse with metadata extraction (title, creator, language, publisher, identifier)
- Unified `SourceDocument` / `SourceChapter` abstraction — TXT and EPUB share the same parse pipeline
- SQLite FTS5 full-text search with Unicode61 tokenizer + CJK keyword fallback
- Hybrid retrieval: FTS + keyword + structured atoms + analysis outputs, dedup, score normalization
- Structured `evidence_json` in chat messages (text, source_type, chunk_id, method, score, locator)
- `RetrievalTrace` per chat/retrieve call for debugging
- Entity evidence: `GET /entities/{id}/evidence` — atoms, chunks, outputs for any entity
- Similar scenes: `GET /similar-scenes` — by chunk_id (builds query seed from atoms+text) or free-text query
- Semantic rerank skeleton: `ENABLE_SEMANTIC_RERANK=False`, `EmbeddingProvider`, `EmbeddingCache` table
- Document metadata endpoint

**Frontend (12 steps):**
- EPUB upload with `.txt,.epub` accept + format description
- Document Metadata Card with file-type badge and EPUB metadata fields
- EPUB Chapter Tree (collapsible, nav_order-sorted, abbreviated source href)
- Source Locator Badge (green for EPUB, gray for TXT, with chapter/chunk info)
- Topic Search Panel with FTS/Keyword Fallback method toggles and inline locator detail
- Retrieval Debug Drawer with method checkboxes for all 5 retrieval methods
- Retrieval Method Badge with 5 distinct colors
- Chat Evidence v3: structured cards with source_type/method/score/title/text, "Open source" for inline locator detail, backward-compatible with legacy `string[]`
- Entity Evidence Explorer: three sections (Atoms / Source Chunks / Related Outputs)
- Similar Scenes Panel: dual-mode (By Query / By Chunk ID) with scored results
- Semantic Rerank UI: disabled checkbox with "(off)" label and tooltip
- UX hardening: `useMemo` optimizations, delete error states, `instanceof Error` guards
- 11 Playwright E2E tests (all mocked, no real LLM needed)
- Updated smoke test document with 8 new v0.3 sections

## Architecture Summary

```
Source Abstraction (v0.3)
  TXT adapter → SourceDocument → SourceChapter
  EPUB adapter → zipfile + xml.etree.ElementTree (OPF/container)
                  + beautifulsoup4 (XHTML text) → SourceDocument → SourceChapter
    │
    ▼
Unified parse pipeline → Chapter rows (source_href, nav_order) + Chunk rows (source_locator_json)
    │
    ▼
FTS Index (chunk_fts virtual table, FTS5 + Unicode61 tokenizer)
    │
    ▼
Hybrid Retrieval
  ├── FTS5 lexical search (BM25)
  ├── Keyword/CJK fallback (LIKE + AND-group char overlap)
  ├── Structured atom search (canonical_name, aliases, evidence_quotes)
  └── Analysis output search (title, content)
    │
    ▼
Dedup by chunk_id → score normalization → top-k
    │
    ├── /retrieve → candidates + optional RetrievalTrace
    ├── /chat → structured evidence_json
    ├── /entities/{id}/evidence → atoms + chunks + outputs
    └── /similar-scenes → scored results (excludes seed chunk)
```

## Known Limitations (v0.3.0)

| Limitation | Detail |
|-----------|--------|
| Semantic rerank disabled | `ENABLE_SEMANTIC_RERANK=False` by default. `EmbeddingProvider` is a skeleton — no real embedding API calls. |
| Final outputs unchanged | v0.3 produces the same 6 output types as v0.2 (overview, characters, relations, events, causality, themes). Worldbuilding and foreshadowing have merge support but no final output builders. |
| .txt and .epub only | PDF, OCR, DRM removal not supported. |
| Single novel per Topic | Multi-book Topics planned for v0.4. |
| No vector DB | FTS5 + keyword + structured retrieval only. `EmbeddingCache` table exists but is unused by default. |
| No streaming | All LLM responses are request/response. |
| Single-user | No authentication or multi-user support. |
| 20 Playwright tests fail | Pre-existing issues: 19 in `analysis-v2.spec.ts`, 1 in `topic-detail.spec.ts` — mock data doesn't trigger v2 analysis panel render. All v0.3 tests pass. |

## Breaking Changes from v0.2.0

- **Database schema**: v0.3 adds `retrieval_trace`, `chunk_fts` (virtual table), and `embedding_cache` tables. v0.2 databases are auto-migrated on startup.
- **Document model**: `file_type` field expanded to `"txt" | "epub"`. `encoding` may be `"epub"` for EPUB documents. `metadata_json` added.
- **Chapter model**: `source_href`, `nav_order`, `metadata_json` fields added.
- **Chunk model**: `source_locator_json` field added.
- **Chat model**: `evidence_json` format changed from `string[]` to structured objects. Backend still reads the old format; frontend `normalizeEvidence()` handles both.
- **API additions**: New endpoints for search, retrieve, locator, entity evidence, and similar scenes. All v0.2 endpoints continue to work.

## Upgrade from v0.2.0

1. `git checkout v0.3.0`
2. Restart backend — `init_db()` auto-migrates the existing SQLite database
3. Restart frontend — `npm install && npm run dev`
4. Existing Topics, Providers, Documents, analysis outputs, and chat sessions are preserved
5. Re-parse EPUB documents if the existing parse predates v0.3 (for source locator metadata)
6. FTS index is rebuilt automatically on parse completion
